# Claude + Obsidian: Стратегия анализа и оптимизации

## Что Claude МОЖЕТ в Obsidian

- Прочитать список плагинов (`community-plugins.json`)
- Найти конфликты/дубли плагинов (напр. Templater vs QuickAdd)
- Проанализировать структуру папок и тегов
- Проверить/оптимизировать скрипты DataviewJS и Templater
- Найти ошибки в кастомных шаблонах

## Ограничения

- Не видит визуальный UX (красиво/некрасиво)
- Не может "покликать" интерфейс
- Только предполагает удобство по коду

## Стратегия "Кради как художник" (GitHub → Vault)

1. **Поиск донора** — найти крутой Vault/плагин на GitHub
2. **Клонирование** — в отдельную папку (не смешивать!)
3. **Анализ (экономно)** — сначала README + `ls -R`, потом точечные файлы
4. **Gap Analysis** — сравнить конкретную фичу донора с моим шаблоном
5. **Чистка** — удалить заменённые "костыли"

## Экономика токенов

| Операция | Стоимость |
|----------|-----------|
| Анализ 1 плагина/шаблона | $0.005–0.02 |
| Анализ репозитория "в лоб" | $1–5 (дорого!) |

### Как не слить бюджет

- `.gitignore` / настройки Roo Code — исключить кэш, картинки, PDF
- Сначала обзор на дешёвой модели (Haiku), потом глубокий анализ (Sonnet)
- Читать не ВСЁ, а точечно по задаче

## Первый промпт для аудита плагинов

> "Проанализируй папку `.obsidian/plugins`. Составь список всех установленных плагинов. Кратко напиши, что делает каждый. Затем посмотри на мои последние 5 заметок и скажи, какие из этих плагинов я вообще не использую (судя по разметке в файлах) и что можно удалить для ускорения работы?"
